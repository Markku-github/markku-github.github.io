# Project Case Study Template

Use this markdown file as a checklist and starter content when creating a new project page from `templates/project-template.html`.

Replace placeholders ({{...}}) in the HTML template with your project's values.

## Metadata
- Project title: {{PROJECT_TITLE}}
- One-line summary: {{ONE_LINE_SUMMARY}}
- Year: {{YEAR}}
- Role: {{ROLE}} (Author / Contributor)
- GitHub URL: {{GITHUB_URL}}

## Sections to fill (suggested content)

### Project Overview
Brief description of the project: goals, audience, scope.

### Challenge
What problem were you solving? Constraints (hardware limits, timing, memory).

### Solution
High-level approach. Algorithms used. Architectural decisions.

### Key Features
- Feature 1
- Feature 2
- Feature 3

### Technical Details / Code snippets
Keep snippets small and focused. Link to full source on GitHub for full context.

### Results & Metrics
Performance numbers, accuracy, power, memory, latency, throughput, etc.

### Lessons Learned
What you would change or keep next time. Trade-offs made.

### Files to include in repo
- schematics/ (images, diagrams)
- docs/ (design notes)
- tests/ (unit or integration tests)

## Checklist before publishing
- [ ] Replace placeholders in HTML template
- [ ] Verify paths (CSS, images) are correct
- [ ] Add 1-3 images/screenshots in `img/` and reference them
- [ ] Add GitHub repo link and any documentation links
- [ ] Proofread and check for sensitive data
